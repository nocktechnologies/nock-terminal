# PRODUCT.md — Nock Terminal

**Register:** product

## Product purpose

Mission control for terminal-native coding agents. Keeps agent sessions, repos, terminals, files, prompts, ports, and notifications visible from one local desktop cockpit. The wedge is local-first agent observability and orchestration, not a generic AI IDE clone.

## Users

Developers (today: Kevin and the Nock agent fleet) running multiple coding agents in parallel — Claude Code, Codex CLI, Gemini CLI, brokered dispatch agents. They live in this app between deep-work stints: glance, launch, attach, dispatch, move on. Repeated daily use by an operator, not a first-time visitor to impress.

## Scene sentence (theme driver)

A developer at a desk in a dim room with several agents working, flicking to the cockpit between tasks to read fleet state in under two seconds, with live terminals (black-background xterm panes) embedded in the same window. Dark theme is structural — the app hosts literal terminals — not a category reflex.

## Tone

Quiet, dense, calm. Operator console, not marketing dashboard. Monospace telemetry, editorial section labels, restrained accents that mean something (cyan = live, amber = warning, green = healthy). Copy is honest to the point of bluntness: capabilities the adapters don't have yet are labeled as future, never implied.

## Anti-references

- Generic AI-IDE clones and "AI magic" gradients
- Marketing-dashboard hero metrics and celebration confetti
- Overclaiming copy (no "attach" buttons that don't truly attach)
- Anything that demands attention instead of rewarding a glance

## Strategic principles

- Visible autonomy: agents work independently; the operator sees state, output, blockers
- Local-first trust: source code stays local unless the user chooses otherwise
- Density is a feature; calm is a constraint
- The product posture (AGENTS.md) is the contract: keep UI claims aligned with adapter truth
